Topic 52: Mitigation (Hacking Wireless Networks)

A robust wireless defense strategy is not about implementing a single silver bullet. It is about building a layered, defense-in-depth posture that addresses technical configurations, physical security, and user awareness. The goal is to make unauthorized access so difficult and detectable that an attacker moves on to an easier target.

Here is a comprehensive set of mitigations, categorized by the attacks they prevent:

**1. Mitigations Against General Unauthorized Access and Password Cracking:**

- **Use WPA3 or Strong WPA2 Configuration:**
  - **WPA3:** This is the current gold standard. It replaces the Pre-Shared Key (PSK) handshake with **Simultaneous Authentication of Equals (SAE)**, which is resistant to offline dictionary attacks. If a client supports WPA3, it should be enforced.
  - **Strong WPA2-PSK:** If WPA3 is not available, use WPA2 with an extremely complex passphrase. The password should be a long, random string of at least 20 characters, stored in a password manager. This makes brute-force attacks computationally infeasible.

- **Implement 802.1X / WPA2/3-Enterprise:** This is the most effective mitigation for organizations. Instead of a single shared password (PSK), each user has unique credentials (username/password or digital certificate). This means:
  - Compromising one user's credentials does not compromise the entire network.
  - Individual users can be easily added or revoked.
  - It requires a backend RADIUS server for authentication.

**2. Mitigations Against Rogue Access Points and Client Attacks:**

- **Wireless Intrusion Detection/Prevention System (WIDS/WIPS):** This is a critical monitoring solution. A WIPS uses dedicated sensors to constantly monitor the radio frequency (RF) environment for:
  - **Rogue APs:** It can detect and automatically contain (block) unauthorized access points, including evil twins.
  - **Misconfigured APs:** It can find company APs with weak security settings.
  - **Ad-hoc Networks:** It can detect peer-to-peer connections between clients.
  - **Known Attack Tools:** It can identify the presence of tools like `aireplay-ng` based on their packet signatures.

- **Client Isolation (Guest Policy):** On guest and public Wi-Fi networks, enable client isolation. This prevents wireless clients from communicating with each other, stopping an attacker who is connected to the same network from scanning or attacking other users' devices.

**3. Mitigations Against Man-in-the-Middle (MitM) and Eavesdropping:**

- **Mandate End-to-End Encryption:** This is the ultimate defense against eavesdropping and MitM.
  - **HTTPS Everywhere:** Enforce the use of HTTPS on all web services. Use HTTP Strict Transport Security (HSTS) to force browsers to use HTTPS.
  - **Corporate VPN:** Require the use of a VPN for any remote or mobile worker accessing company resources. A VPN creates an encrypted tunnel from the user's device to the corporate network, making all traffic unreadable to a MitM attacker, even on a rogue AP.

- **Physical Security:** Control physical access to areas where wireless access points are installed. An attacker cannot plug a rogue AP into your network jack if they cannot get into your building.

**4. General Wireless Hygiene and Monitoring:**

- **Regular Audits and Penetration Testing:** Periodically hire ethical hackers to perform wireless penetration tests against your own network. They will use the exact tools and techniques we've discussed to find weaknesses before malicious actors do.
- **SSID Management:**
  - **Disable SSID Broadcast for sensitive networks:** This doesn't make the network "hidden" (as it can still be discovered), but it reduces its visibility to casual scanners.
  - **Use Uncommon SSID Names:** Avoid using default or company-name-only SSIDs. This makes it slightly harder for an attacker to create a convincing evil twin.
- **Strict Access Point Management:** Ensure all corporate APs are running the latest firmware, have strong administrative passwords, and have unnecessary services (like Telnet) disabled.

A secure wireless network is a vigilant one. It combines strong encryption, unique user authentication, continuous monitoring, and an educated user base to create a resilient environment.

Glossary for Topic 52:

WPA3: The latest Wi-Fi security protocol, which provides stronger encryption and protection against offline password cracking.

WPA2/3-Enterprise: A mode of WPA that requires a RADIUS server for authentication, providing individual user logins instead of a shared password.

RADIUS Server: A central server that provides authentication, authorization, and accounting (AAA) management for users who connect to a network.

WIPS (Wireless Intrusion Prevention System): A network security device that monitors the radio spectrum for unauthorized access points and can automatically take countermeasures.

Client Isolation: A feature on wireless access points that prevents wireless clients from communicating with each other.

VPN (Virtual Private Network): A service that extends a private network across a public network, enabling users to send and receive data as if their devices were directly connected to the private network, with strong encryption.

HSTS (HTTP Strict Transport Security): A web security policy mechanism that helps to protect websites against protocol downgrade attacks and cookie hijacking.

Mitigation: An action taken to reduce the severity, impact, or likelihood of a security threat.
