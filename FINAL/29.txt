Topic 29: Tools for Password Hacking

A craftsman is only as good as their tools. In password cracking, the tools are the engines that perform the computationally intensive work of generating and testing billions of password guesses. The two most dominant and powerful tools in this space are **Hashcat** and **John the Ripper**. Understanding their strengths, weaknesses, and use cases is essential for any professional.

**Hashcat: The Speed King**

Hashcat bills itself as the world's fastest password cracker. It is a CPU-based cracker, but its true power is unleashed when using a GPU (Graphics Processing Unit). GPUs have thousands of cores designed for parallel processing, making them vastly superior to CPUs for the repetitive task of computing hashes. Hashcat is the tool of choice for serious, high-speed cracking.

- **Key Features:**
  - **Multi-Platform:** Runs on Windows, Linux, and macOS.
  - **Multi-Algorithm:** Supports hundreds of hash types (MD5, SHA1, NTLM, bcrypt, WPA2, etc.).
  - **Multiple Attack Modes:** It has highly optimized implementations of all the major attack types we discussed (dictionary, combination, brute-force/mask, hybrid).
  - **GPU Acceleration:** Its primary advantage, leveraging the power of your graphics card.

- **Common Hashcat Command Structure:**
  `hashcat -m [Hash-Type] -a [Attack-Mode] [Hashfile] [Wordlist|Mask]`
  - `-m 0` for MD5
  - `-m 1000` for NTLM (Windows)
  - `-a 0` for a straight dictionary attack.
  - `-a 3` for a brute-force/mask attack.

- **Example Command (Dictionary Attack on NTLM hashes):**
  `hashcat -m 1000 -a 0 -o cracked_ntlm.txt ntlm_hashes.txt /usr/share/wordlists/rockyou.txt`

**John the Ripper: The Versatile Veteran**

John the Ripper (often just "John") is an older, highly versatile password cracker. It's known for its community support, its powerful rule-based engine, and the fact that it's a standard part of every penetration tester's toolkit. While it can be slower than Hashcat for raw brute-forcing, its strength lies in its intelligence and flexibility, especially with its rule-based attacks.

- **Key Features:**
  - **Strong Rule-Based Engine:** John's "single crack" mode and its extensive rule sets are incredibly effective at cracking human-chosen passwords.
  - **Community Support:** Has a wide range of community-developed patches and scripts for different hash types.
  - **"Out of the Box" Functionality:** It often works with minimal configuration and can automatically detect many common hash types.

- **Common John the Ripper Command Structure:**
  `john [Options] [Hashfile]`
  - `--wordlist=[list]` to specify a wordlist.
  - `--format=[format]` to specify the hash type (e.g., `raw-md5`, `nt`).
  - `--rules` to enable the powerful rule-based mangling.

- **Example Command (Rule-Based Attack on MD5 hashes):**
  `john --format=raw-md5 --wordlist=rockyou.txt --rules target_hashes.txt`

**Other Important Tools:**

- **Hydra:** While Hashcat and John are used for cracking *offline* password hashes, Hydra is used for conducting *online* password attacks. It performs rapid password guessing against various network services like SSH, FTP, HTTP forms, and RDP.
  - **Example Command (SSH Brute-Force):**
    `hydra -l username -P passwords.txt ssh://192.168.1.100`

- **Crunch:** A wordlist generator. You can use Crunch to generate custom wordlists based on specific character sets and patterns. This is useful for creating targeted wordlists for brute-force or hybrid attacks.
  - **Example Command (Generate all 6-character passwords with lowercase letters):**
    `crunch 6 6 abcdefghijklmnopqrstuvwxyz -o custom_wordlist.txt`

**Choosing the Right Tool:**

- **Use Hashcat when:** You have a powerful GPU and are cracking a large number of hashes using fast algorithms (like NTLM, MD5). You need raw speed.
- **Use John the Ripper when:** You want to leverage its intelligent rule sets for a more "human" approach to cracking, or when you are working on a system without a strong GPU.
- **Use Hydra when:** You are performing an online attack against a live service and need to test login credentials directly.

A professional will often use these tools in tandem. They might use John with rules for an initial pass, and if that fails, switch to Hashcat with a GPU for a massive brute-force attack.

Glossary for Topic 29:

Hashcat: A high-speed password recovery tool that leverages the power of the GPU to crack password hashes.

John the Ripper: A versatile, feature-rich password cracking tool known for its powerful rule-based attack modes.

Hydra: A parallelized login cracker which supports numerous protocols to attack online services.

Crunch: A wordlist generator where you can specify a standard character set or a character set you specify.

GPU (Graphics Processing Unit): A specialized electronic circuit designed to rapidly manipulate and alter memory to accelerate the creation of images, but also highly effective for password cracking due to its parallel architecture.

Attack Mode: In Hashcat, a number that defines the cracking methodology (e.g., 0 for straight, 3 for brute-force).

Rule-Based Engine: A system within a password cracker that automatically mutates words from a wordlist according to predefined patterns to simulate common human password creation habits.
